## Git］その時点でのファイルの中身を確認したい

```bash
git show <コミットハッシュ> -- "パス/ファイル名"
```

## Git］特定のファイルがいつ消されたのか知りたい

ファイルを間違って消してしまったっぽいけど、いつ消したか覚えてないとき。

```bash
git log --all --full-history -- "パス/ファイル名"
```

そのファイルに関わるコミット一覧が表示される。
一番上（最新）のコミットが削除したコミット。

## Bash］一時ディレクトリ処理のベストプラクティスパターン

- #storyline/proper7y

一時ディレクトリを使った処理をしたいとき。

```bash
do_something_with_temp_dir() {
  (
    local TEMP_DIR
    TEMP_DIR="$(mktemp -d)"
    trap 'rm -rf "${TEMP_DIR:-}"' EXIT

    cd "$TEMP_DIR"
    # ... download, extract, copy, etc. ...
  )
  # Post-subshell: update any global state here
}
```

- `mktemp -d`: 衝突しないユニークな一時ディレクトリをアトミックに作成する
- `trap '...' EXIT`: 成功・失敗問わずクリーンアップを保証する
- `( )`: trap のスコープを関数内に限定し、プロセス全体の trap を汚染しない
- `${TEMP_DIR:-}`: 変数未定義時の `rm -rf ""` を防ぐ防衛的プログラミング
- グローバル変数の更新をサブシェルの外に置く: サブシェル内の変更は親に伝わらないため

## Bash］「サブシェル」とは何か

- #storyline/proper7y

- いま動いているシェルとは別に作られる一時的なシェル実行環境
- `( ... )` のように処理を括弧で囲むことで、処理を閉じ込める
- よくある使い方：
  - 一時的に作業ディレクトリを変えて処理する
  - 途中の環境変更を外へ漏らしたくない処理をまとめる
- 注意点：
  - サブシェル内の変数変更・cd は親シェルに伝播しない
  - ただし、`export` した変数は伝播するなどの例外あり
